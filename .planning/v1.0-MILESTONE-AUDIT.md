## Critical Integration Gaps

### 1. BROKEN: Product Search UI → API Route Mismatch

**Requirements Affected:**
- SEARCH-01: Agency admin can search products within a single client's catalog
- SEARCH-07: Search results only include products from selected client store (tenant isolation)
- UI-07: Agency admin can search products within a client's catalog

**Issue:**
Dashboard product search page calls incorrect API endpoint

**Location:** `public/js/dashboard.js:467`

**Current Code:**
```javascript
const response = await fetch(`/api/v1/tenants/${this.tenantId}/products?query=...`, {
```

**Should Be:**
```javascript
const response = await fetch(`/api/v1/tenants/${this.tenantId}/search?query=...`, {
```

**API Route Exists:** `GET /api/v1/tenants/{tenantId}/search` (ProductSearchController)

**Impact:**
- Product search functionality **completely broken** in dashboard
- Users cannot search products despite:
  - Phase 5 Elasticsearch integration complete ✅
  - ProductSearchController implemented ✅
  - Route properly registered ✅
  - Dashboard UI exists ✅

**Status:** 🔴 **UNWIRED** (Backend complete, frontend calling wrong endpoint)

**Fix Required:** 1-line change in `public/js/dashboard.js:467`

---

### 2. BROKEN: Sync Trigger Button → Missing API Route

**Requirements Affected:**
- SYNC-01: Agency admin can trigger manual catalog sync for a specific client store
- UI-05: Agency admin can trigger sync operation for each client store

**Issue:**
Dashboard has "Trigger Sync" button but calls non-existent route

**Location:** `public/js/dashboard.js:181`

**Current Code:**
```javascript
const response = await fetch(`/api/v1/tenants/${tenantId}/sync`, {
    method: 'POST',
```

**Problem:**
No route exists at `/api/v1/tenants/{tenantId}/sync`

**Actual Sync Route:** `POST /api/v1/sync/dispatch` (requires `tenant_id` in request body, not URL)

**Backend Implementation:**
SyncController@dispatch expects:
```json
{
    "tenant_id": "uuid",
    "data": {}
}
```

**Impact:**
- Sync trigger functionality **completely broken**
- Despite:
  - Phase 6 sync jobs complete ✅
  - Phase 4 queue infrastructure operational ✅
  - SyncController exists with dispatch method ✅
  - Dashboard UI has trigger button ✅

**Status:** 🔴 **UNWIRED** (Backend complete, frontend calling wrong endpoint pattern)

**Fix Required:** Change dashboard to call `/api/v1/sync/dispatch` with `tenant_id` in body, OR add route alias

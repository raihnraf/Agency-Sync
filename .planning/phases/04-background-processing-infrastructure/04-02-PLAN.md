    Create job middleware for automatic tenant context restoration:

    1. Create app/Queue/Middleware/SetTenantContext.php:
       ```php
       <?php

       namespace App\Queue\Middleware;

       use Closure;
       use App\Models\Tenant;

       class SetTenantContext
       {
           public function handle(object $job, Closure $next): void
           {
               if (property_exists($job, 'tenantId')) {
                   $tenant = Tenant::findOrFail($job->tenantId);
                   app()->instance('currentTenant', $tenant);
                   Tenant::setCurrentTenant($tenant);
               }

               $next($job);

               // Clear tenant context after job
               app()->forgetInstance('currentTenant');
               Tenant::setCurrentTenant(null);
           }
       }
       ```

    2. Register middleware alias in bootstrap/app.php (if not already):
       ```php
       Queue::failing(function (JobProcessingFailed $event) {
           \App\Services\QueueJobTracker::markAsFailed($event->job, $event->exception);
       });
       ```

    Implementation notes:
    - Uses property_exists() to check for tenantId (flexible for different job types)
    - Tenant::findOrFail() throws 404 if tenant deleted (fail fast)
    - Stores tenant in app()->instance() for dependency injection
    - Calls Tenant::setCurrentTenant() for global scopes (from Phase 3)
    - Cleans up context after job to prevent memory leaks

    Reference RESEARCH.md Pattern 2 and Phase 3 tenant middleware patterns.

    Create service for tracking job lifecycle:

    1. Create app/Services/QueueJobTracker.php:
       ```php
       <?php

       namespace App\Services;

       use App\Models\JobStatus;
       use Illuminate\Contracts\Queue\Job;

       class QueueJobTracker
       {
           public function track(Job $job, string $jobType): JobStatus
           {
               return JobStatus::create([
                   'job_id' => $job->getJobId(),
                   'tenant_id' => property_exists($job, 'tenantId') ? $job->tenantId : null,
                   'job_type' => $jobType,
                   'status' => 'pending',
                   'payload' => $job->payload(),
               ]);
           }

           public function markAsRunning(Job $job): void
           {
               $status = JobStatus::where('job_id', $job->getJobId())->first();
               if ($status) {
                   $status->markAsRunning();
               }
           }

           public function markAsCompleted(Job $job): void
           {
               $status = JobStatus::where('job_id', $job->getJobId())->first();
               if ($status) {
                   $status->markAsCompleted();
               }
           }

           public function markAsFailed(Job $job, \Throwable $exception): void
           {
               $status = JobStatus::where('job_id', $job->getJobId())->first();
               if ($status) {
                   $status->markAsFailed($exception->getMessage());
               }
           }
       }
       ```

    2. Register queue event listeners in app/Providers/AppServiceProvider.php:
       ```php
       <?php

       namespace App\Providers;

       use Illuminate\Support\ServiceProvider;
       use Illuminate\Queue\Events\JobProcessing;
       use Illuminate\Queue\Events\JobProcessed;
       use Illuminate\Queue\Events\JobFailed;
       use App\Services\QueueJobTracker;

       class AppServiceProvider extends ServiceProvider
       {
           public function boot(): void
           {
               // Track job lifecycle
               \Queue::before(function (JobProcessing $event) {
                   app(QueueJobTracker::class)->markAsRunning($event->job);
               });

               \Queue::after(function (JobProcessed $event) {
                   if ($event->job->hasFailed()) {
                       return; // Handled by failing callback
                   }
                   app(QueueJobTracker::class)->markAsCompleted($event->job);
               });

               \Queue::failing(function (JobFailed $event) {
                   app(QueueJobTracker::class)->markAsFailed($event->job, $event->exception);
               });
           }
       }
       ```

    Implementation notes:
    - before event: Job picked up by worker (mark running)
    - after event: Job completed successfully (mark completed)
    - failing event: Job failed after all retries (mark failed)
    - Service automatically extracts tenantId from job payload
    - Graceful handling if JobStatus record doesn't exist

    Reference RESEARCH.md job lifecycle tracking.

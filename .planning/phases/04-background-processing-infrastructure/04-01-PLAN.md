<task type="auto">
  <name>Task 2: Create Supervisor configuration for Laravel queue workers</name>
  <files>docker/supervisor/supervisord.conf, docker-compose.yml</files>
  <action>
    Create Supervisor configuration to monitor Laravel queue workers:

    1. Create docker/supervisor/supervisord.conf:
       ```ini
       [supervisord]
       nodaemon=true
       user=root
       logfile=/var/log/supervisor/supervisord.log
       pidfile=/var/run/supervisord.pid

       [program:laravel-worker]
       process_name=%(program_name)s_%(process_num)02d
       command=php /var/www/html/artisan queue:work redis --sleep=3 --tries=3 --max-time=3600 --timeout=120
       autostart=true
       autorestart=true
       stopasgroup=true
       killasgroup=true
       user=www-data
       numprocs=2
       redirect_stderr=true
       stdout_logfile=/var/www/html/storage/logs/worker.log
       stopwaitsecs=3600
       ```

    2. Update docker-compose.yml app service:
       - Build Supervisor into the image (install via apt-get)
       - Copy supervisord.conf into container
       - Change CMD to run supervisord instead of php-fpm
       - Ensure php-fpm runs under Supervisor

    3. Add to Dockerfile (or update if exists):
       ```dockerfile
       RUN apt-get update && apt-get install -y supervisor \
           && mkdir -p /var/log/supervisor
       COPY docker/supervisor/supervisord.conf /etc/supervisor/conf.d/supervisord.conf
       CMD ["/usr/bin/supervisord", "-n", "-c", "/etc/supervisor/conf.d/supervisord.conf"]
       ```

    Configuration details:
    - numprocs=2: Start 2 worker processes (adjustable for production)
    - --sleep=3: Wait 3 seconds between jobs (prevent CPU spinning)
    - --tries=3: Retry failed jobs 3 times (matches requirement QUEUE-05)
    - --timeout=120: Kill jobs running > 2 minutes
    - --max-time=3600: Worker restarts after 1 hour (memory leak prevention)
    - autorestart=true: Auto-restart workers on crash
    - stopasgroup=true, killasgroup=true: Clean shutdowns
  </action>
  <verify>
    <automated>docker compose up -d --build app && sleep 5 && docker compose exec app supervisorctl status | grep "laravel-worker.*RUNNING"</automated>
  </verify>
  <done>
    - docker/supervisor/supervisord.conf exists with laravel-worker program
    - Dockerfile installs supervisor and copies config
    - docker-compose.yml app service runs supervisord as CMD
    - supervisorctl status shows laravel-worker processes in RUNNING state
  </done>
</task>

    ```ini
    redirect_stderr=true

    process_name=%(program_name)s_%(process_num)02d
    command=php /var/www/html/artisan queue:work redis --sleep=3 --tries=3 --max-time=3600 --timeout=120
    autostart=true
    autorestart=true
    stopasgroup=true
    killasgroup=true
    user=www-data
    numprocs=2
    redirect_stderr=true
    stdout_logfile=/var/www/html/storage/logs/worker.log
    stopwaitsecs=3600
    ```

    This ensures:
    - PHP-FPM runs under Supervisor (same process lifecycle)
    - Queue workers auto-restart on failure
    - Both services managed by single supervisord instance
    - Logs written to separate files for debugging

    Update docker-compose.yml to remove php-fpm specific configs if any (now managed by Supervisor).
  </action>

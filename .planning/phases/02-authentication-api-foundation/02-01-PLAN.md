Create AuthController with registration, login, and logout endpoints following Laravel Sanctum patterns.

Create app/Http/Controllers/Api/V1/AuthController.php:
- Namespace: App\Http\Controllers\Api\V1
- Extends Controller
- Constructor: (empty for now)

register(RegisterRequest $request):
- Validate via RegisterRequest (automatic)
- Create user: User::create([...validated fields, Hash::make(password)])
- Create token: $user->createToken('api-token', ['*'], now()->addHours(4))
- Return response: {data: {user: UserResource, token: plainTextToken}, meta: {expires_at}}, status 201

login(LoginRequest $request):
- Find user by email
- Verify password with Hash::check()
- If invalid: return {errors: [{field: 'email', message: 'Invalid credentials'}]}, status 401
- Create token: $user->createToken('api-token', ['*'], now()->addHours(4))
- Return response: {data: {user: UserResource, token: plainTextToken}, meta: {expires_at}}, status 200

logout(Request $request):
- Get current token: $request->user()->currentAccessToken()
- Delete token: $token->delete()
- Return response: null, status 204

Per CONTEXT.md decisions:
- "4-hour duration — Tokens valid for 4 hours during active use"
- "Logout invalidates token — Token deletion on logout"
- "Field-based validation errors"

Per RESEARCH.md patterns:
- Use Hash facade for password verification
- Use currentAccessToken() for logout (revokes only current device)
- Return consistent JSON structure with data/meta/error fields

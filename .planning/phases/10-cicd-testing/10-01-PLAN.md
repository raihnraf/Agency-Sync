<task type="auto">
  <name>Task 1: Create GitHub Actions CI workflow</name>
  <files>.github/workflows/ci.yml</files>
  <action>
Create GitHub Actions workflow file at `.github/workflows/ci.yml`:

```yaml
name: CI

on:
  push:
    branches: [ main, develop ]
  pull_request:
    branches: [ main ]

jobs:
  test:
    runs-on: ubuntu-latest

    services:
      mysql:
        image: mysql:8.0
        env:
          MYSQL_ROOT_PASSWORD: password
          MYSQL_DATABASE: agency_sync_test
        ports:
          - 3306:3306
        options: >-
          --health-cmd="mysqladmin ping"
          --health-interval=10s
          --health-timeout=5s
          --health-retries=3

    steps:
      - name: Checkout code
        uses: actions/checkout@v4

      - name: Setup PHP
        uses: shivammathur/setup-php@v2
        with:
          php-version: '8.2'
          extensions: mbstring, pdo, pdo_mysql, redis, bcmath, gd, zip
          coverage: xdebug

      - name: Get Composer Cache Directory
        id: composer-cache
        run: echo "dir=$(composer config cache-files-dir)" >> $GITHUB_OUTPUT

      - name: Cache Composer dependencies
        uses: actions/cache@v4
        with:
          path: ${{ steps.composer-cache.outputs.dir }}
          key: ${{ runner.os }}-composer-${{ hashFiles('**/composer.lock') }}
          restore-keys: ${{ runner.os }}-composer-

      - name: Install Dependencies
        run: composer install --no-interaction --prefer-dist --optimize-autoloader

      - name: Copy .env
        run: cp .env.example .env

      - name: Generate Key
        run: php artisan key:generate

      - name: Clear Config Cache
        run: php artisan config:clear

      - name: Run Tests with Coverage
        run: vendor/bin/phpunit --coverage-clover=coverage.xml --coverage-text

      - name: Upload Coverage Reports
        uses: actions/upload-artifact@v4
        with:
          name: coverage-report
          path: coverage.xml
```

**Key implementation details:**
- Triggers on push to main/develop and PRs to main
- Uses MySQL 8.0 service container with health checks
- PHP 8.2 with required extensions (mbstring, pdo, pdo_mysql, redis, bcmath, gd, zip)
- Xdebug enabled for coverage (coverage: xdebug)
- Composer caching for faster builds
- Runs PHPUnit with coverage clover and text output
- Uploads coverage.xml as artifact for visibility

<task type="auto">
  <name>Task 2: Configure PHPUnit coverage threshold</name>
  <files>phpunit.xml</files>
  <action>
Update phpunit.xml to add coverage configuration with 70% threshold:

**After the <source> section, add:**

```xml
    <coverage>
        <report>
            <html outputDirectory="coverage/html"/>
            <text outputFile="php://stdout" showUncoveredFiles="false" showOnlySummary="true"/>
            <clover outputFile="coverage/clover.xml"/>
        </report>
        <threshold>
            <global>
                <line value="70"/>
                <method value="70"/>
            </global>
        </threshold>
    </coverage>
```

**Also add failOnRisky and failOnWarning attributes:**
Update the opening <phpunit> tag to include:
```xml
<phpunit xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:noNamespaceSchemaLocation="vendor/phpunit/phpunit/phpunit.xsd"
         bootstrap="vendor/autoload.php"
         colors="true"
         cacheDirectory=".phpunit.cache"
         testdox="true"
         executionOrder="default"
         failOnRisky="true"
         failOnWarning="true">
```

**Key implementation details:**
- <coverage> section configures coverage reporting
- Three report formats: HTML (local viewing), text (console), clover (CI)
- <threshold> enforces 70% minimum coverage for lines and methods
- failOnRisky and failOnWarning ensure test quality
- HTML reports go to coverage/html directory (gitignored)
- Clover XML goes to coverage/clover.xml (uploaded by CI)
- Text output goes to stdout for visibility in CI logs

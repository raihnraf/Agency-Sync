  <behavior>

    - Test 2: Missing X-Tenant-ID header returns 422 with explicit error
    - Test 3: Valid tenant ID that belongs to user sets tenant context
    - Test 4: Invalid tenant ID (not found or user not associated) returns 404 with generic error
    - Test 5: Tenant stored in request attributes and user model via setCurrentTenant()
  </behavior>
  <action>
Create SetTenant middleware:
- Extract X-Tenant-ID header from request
- If missing, return 422 with {errors: [{field: 'X-Tenant-ID', message: 'X-Tenant-ID header is required'}]}
- Query Tenant model: Tenant::where('id', $tenantId)->whereHas('users', fn($q) => $q->where('user_id', $request->user()->id))->first()
- If not found, return 404 with {errors: [{message: 'Tenant not found or access denied'}]} (generic to prevent enumeration)
- Store tenant in request attributes: $request->attributes->set('current_tenant', $tenant)
- Set tenant on user: auth()->user()->setCurrentTenant($tenant)
- Return $next($request)

Create SetTenantTest unit test:
- Test missing header returns 422
- Test invalid tenant ID returns 404
- Test valid tenant ID sets context
- Test generic error message prevents enumeration
  </action>
  <verify>
    <automated>php artisan test tests/Unit/Middleware/SetTenantTest.php</automated>
  </verify>
  <done>
SetTenant middleware created with header-based tenant resolution. All tests pass proving tenant context is set correctly and enumeration is prevented.
  </done>

<task type="auto">
  <name>Task 3: Configure Scribe for Sanctum authentication</name>
  <files>config/scribe.php</files>
  <action>
    Configure Scribe to use Laravel Sanctum for authenticated endpoints:

    Edit config/scribe.php, modify 'auth' section:
    - Set 'enabled' => true
    - Set 'default' => 'sanctum'
    - Configure 'sanctum' array:
      * 'type' => 'bearer' (Sanctum uses Bearer tokens)
      * 'name' => 'Authorization' (HTTP header name)
      * 'use_test_user' => true (enable test user for interactive docs)
      * 'test_user' => [
          'email' => env('SCRIBE_TEST_USER_EMAIL', 'admin@agencysync.com'),
          'password' => env('SCRIBE_TEST_USER_PASSWORD', 'password'),
        ]

    Configure 'routes' section:
    - Set 'match.domains' => ['*'] (all domains)
    - Set 'match.prefixes' => ['api/*'] (only API routes)
    - Set 'match.exclude' => ['api/v1/health', 'api/v1/internal/*'] (exclude internal routes)

    This enables interactive "Try it out" buttons in documentation with auto-injected Sanctum tokens.
  </action>
  <verify>
    <automated>php artisan config:cache && grep -q "sanctum" config/scribe.php && echo "Sanctum configured"</automated>
  </verify>
  <done>Scribe configured with Sanctum auth, test user credentials set for interactive docs</done>
</task>

<task type="auto">
  <name>Task 4: Register /docs route in web routes</name>
  <files>routes/web.php</files>
  <action>
    Add /docs route to routes/web.php for viewing documentation:

    Route placement:

       - Volumes: bind mount ./ to /var/www, ./docker/nginx/default.conf to /etc/nginx/conf.d/default.conf:ro
       - Networks: frontend (exposed) and backend (internal)

       - Image: mysql:8.0
       - Container name: agency-sync-mysql
       - Environment: MYSQL_ROOT_PASSWORD, MYSQL_DATABASE, MYSQL_USER, MYSQL_PASSWORD (with ${VAR:-default} syntax)

       - Networks: backend only (internal for security)
       - Healthcheck: mysqladmin ping -h localhost (interval 10s, timeout 5s, retries 5)

    4. **redis service**:
       - Image: redis:7-alpine
       - Container name: agency-sync-redis
       - Command: redis-server --appendonly yes (persistence enabled)
       - Volumes: named volume redis-data:/data
       - Networks: backend only (internal for security)
       - Healthcheck: redis-cli ping (interval 10s, timeout 5s, retries 5)

       - Image: elasticsearch:8.13.0
       - Container name: agency-sync-elasticsearch

       - Healthcheck: curl -sf http://localhost:9200/_cluster/health (interval 30s, timeout 10s, retries 5)

    6. **Networks**:
       - frontend: driver bridge (exposed to host)
       - backend: driver bridge with internal: true (isolated from external network)

    7. **Volumes**:
       - mysql-data: driver local
       - redis-data: driver local

<task type="auto">
  <name>Task 2: Create PHP-FPM Dockerfile with Laravel extensions</name>
  <files>Dockerfile</files>
  <action>
    Create Dockerfile in project root with the following:

    1. **Base image**: FROM php:8.2-fpm

    2. **System dependencies**:
       - Install git, curl, libpng-dev, libonig-dev, libxml2-dev, zip, unzip, libicu-dev, libzip-dev
       - Clean up apt cache to keep image small

    3. **PHP extensions** (required for Laravel 11):
       - docker-php-ext-install: pdo_mysql, mbstring, exif, pcntl, bcmath, gd, intl, zip

    4. **Redis extension**:
       - pecl install redis && docker-php-ext-enable redis

    5. **Composer installation**:
       - COPY --from=composer:latest /usr/bin/composer /usr/bin/composer

    6. **Working directory**: WORKDIR /var/www

    7. **Copy application**: COPY . /var/www

    8. **Permissions** (critical for Laravel):
       - chown -R www-data:www-data /var/www
       - chmod -R 775 /var/www/storage
       - chmod -R 775 /var/www/bootstrap/cache

    9. **Startup script**: COPY docker/php/docker-entrypoint.sh /usr/local/bin/docker-entrypoint.sh && chmod +x

    10. **Entrypoint/CMD**: ENTRYPOINT ["docker-entrypoint.sh"] and CMD ["php-fpm"]

    Reference patterns from RESEARCH.md "Pattern 2: PHP-FPM Dockerfile for Laravel 11".
  </action>
  <verify>
    <automated>docker compose build app 2>&1 | grep -q "Successfully built" && echo "Dockerfile build successful"</automated>
  </verify>
  <done>Dockerfile exists, builds successfully, includes all required PHP extensions and Composer</done>
</task>

<task type="auto">
  <name>Task 3: Create Nginx virtual host configuration</name>
  <files>docker/nginx/default.conf</files>
  <action>
    Create docker/nginx/default.conf with the following:

    1. **Server block**:
       - listen 80
       - server_name localhost
       - root /var/www/public
       - index index.php index.html

    2. **PHP-FPM proxy** (critical for Laravel):
       - location ~ \.php$ block
       - try_files $uri =404
       - fastcgi_split_path_info ^(.+\.php)(/.+)$
       - fastcgi_pass app:9000 (service name from compose.yaml)
       - fastcgi_index index.php
       - include fastcgi_params
       - fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name
       - fastcgi_param PATH_INFO $fastcgi_path_info

    3. **Laravel routing**:
       - location / block
       - try_files $uri $uri/ /index.php?$query_string
       - gzip_static on

    4. **Security**:
       - location ~ /\. block with deny all (prevent access to .env, .git, etc.)

    5. **Logging**:
       - access_log /var/log/nginx/access.log
       - error_log /var/log/nginx/error.log

    Reference patterns from RESEARCH.md "Pattern 3: Nginx Virtual Host Configuration".
  </action>
  <verify>
    <automated>docker compose run --rm nginx nginx -t 2>&1 | grep -q "syntax is ok" && echo "Nginx config valid"</automated>
  </verify>
  <done>docker/nginx/default.conf exists with valid syntax, proxies PHP to app:9000</done>
</task>

<task type="auto">
  <name>Task 4: Create PHP-FPM container startup script</name>
  <files>docker/php/docker-entrypoint.sh</files>
  <action>
    Create docker/php/docker-entrypoint.sh with the following:

    1. **Shebang**: #!/bin/bash

    2. **Set permissions** (run as container starts):
       - chown -R www-data:www-data /var/www/storage /var/www/bootstrap/cache (ensure writable on each start)

    3. **Execute main process**: exec php-fpm (use exec to replace shell, PID 1)

    The script should be executable (chmod +x) and handle permission fixes before starting PHP-FPM.

    This solves the common "Permission denied" issue when containers start with mismatched UID/GID between host and container.
  </action>

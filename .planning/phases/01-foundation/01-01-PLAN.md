  <files>compose.yaml</files>
  <action>
    Create compose.yaml (Docker Compose v2 format) with the following services:

    1. **app service** (PHP-FPM):
       - Build from Dockerfile in project root
       - Container name: agency-sync-app
       - Working directory: /var/www
       - Volumes: bind mount project ./ to /var/www, anonymous volume /var/www/vendor (prevents mount issues)
       - Environment: APP_ENV, DB_CONNECTION, DB_HOST=mysql, DB_PORT=3306, DB_DATABASE, DB_USERNAME, DB_PASSWORD, REDIS_HOST=redis, REDIS_PORT=6379, ELASTICSEARCH_HOST=elasticsearch, ELASTICSEARCH_PORT=9200
       - Networks: backend only (internal)
       - Depends on: mysql, redis, elasticsearch with condition: service_healthy

    2. **nginx service**:
       - Image: nginx:1.25-alpine
       - Container name: agency-sync-nginx
       - Ports: "${NGINX_PORT:-80}:80"
       - Volumes: bind mount ./ to /var/www, ./docker/nginx/default.conf to /etc/nginx/conf.d/default.conf:ro
       - Networks: frontend (exposed) and backend (internal)
       - Depends on: app service

    3. **mysql service**:
       - Image: mysql:8.0
       - Container name: agency-sync-mysql
       - Environment: MYSQL_ROOT_PASSWORD, MYSQL_DATABASE, MYSQL_USER, MYSQL_PASSWORD (with ${VAR:-default} syntax)
       - Volumes: named volume mysql-data:/var/lib/mysql
       - Networks: backend only (internal for security)
       - Healthcheck: mysqladmin ping -h localhost (interval 10s, timeout 5s, retries 5)

    4. **redis service**:
       - Image: redis:7-alpine
       - Container name: agency-sync-redis
       - Command: redis-server --appendonly yes (persistence enabled)
       - Volumes: named volume redis-data:/data
       - Networks: backend only (internal for security)
       - Healthcheck: redis-cli ping (interval 10s, timeout 5s, retries 5)

    5. **elasticsearch service**:
       - Image: elasticsearch:8.13.0
       - Container name: agency-sync-elasticsearch
       - Environment: discovery.type=single-node, xpack.security.enabled=false, ES_JAVA_OPTS=-Xms512m -Xmx512m, bootstrap.memory_lock=true
       - ulimits: memlock soft=-1 hard=-1
       - Volumes: named volume es-data:/usr/share/elasticsearch/data
       - Networks: backend only (internal for security)
       - Healthcheck: curl -sf http://localhost:9200/_cluster/health (interval 30s, timeout 10s, retries 5)

    6. **Networks**:
       - frontend: driver bridge (exposed to host)
       - backend: driver bridge with internal: true (isolated from external network)

    7. **Volumes**:
       - mysql-data: driver local
       - redis-data: driver local
       - es-data: driver local

    Use ${VAR:-default} syntax for all environment variables to provide sensible defaults.
    Reference patterns from RESEARCH.md "Pattern 1: Docker Compose v2 Service Definition".

  <files>Dockerfile</files>
  <action>
    Create Dockerfile in project root with the following:

    1. **Base image**: FROM php:8.2-fpm

    2. **System dependencies**:
       - Install git, curl, libpng-dev, libonig-dev, libxml2-dev, zip, unzip, libicu-dev, libzip-dev
       - Clean up apt cache to keep image small

    3. **PHP extensions** (required for Laravel 11):
       - docker-php-ext-install: pdo_mysql, mbstring, exif, pcntl, bcmath, gd, intl, zip

    4. **Redis extension**:
       - pecl install redis && docker-php-ext-enable redis

    5. **Composer installation**:
       - COPY --from=composer:latest /usr/bin/composer /usr/bin/composer

    6. **Working directory**: WORKDIR /var/www

    7. **Copy application**: COPY . /var/www

    8. **Permissions** (critical for Laravel):
       - chown -R www-data:www-data /var/www
       - chmod -R 775 /var/www/storage
       - chmod -R 775 /var/www/bootstrap/cache

    9. **Startup script**: COPY docker/php/docker-entrypoint.sh /usr/local/bin/docker-entrypoint.sh && chmod +x

    10. **Entrypoint/CMD**: ENTRYPOINT ["docker-entrypoint.sh"] and CMD ["php-fpm"]

    Reference patterns from RESEARCH.md "Pattern 2: PHP-FPM Dockerfile for Laravel 11".

<task type="auto">
  <name>Task 3: Create Nginx virtual host configuration</name>
  <files>docker/nginx/default.conf</files>
  <action>
    Create docker/nginx/default.conf with the following:

    1. **Server block**:
       - listen 80
       - server_name localhost
       - root /var/www/public
       - index index.php index.html

    2. **PHP-FPM proxy** (critical for Laravel):
       - location ~ \.php$ block
       - try_files $uri =404
       - fastcgi_split_path_info ^(.+\.php)(/.+)$
       - fastcgi_pass app:9000 (service name from compose.yaml)
       - fastcgi_index index.php
       - include fastcgi_params
       - fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name
       - fastcgi_param PATH_INFO $fastcgi_path_info

    3. **Laravel routing**:
       - location / block
       - try_files $uri $uri/ /index.php?$query_string
       - gzip_static on

    4. **Security**:
       - location ~ /\. block with deny all (prevent access to .env, .git, etc.)

    5. **Logging**:
       - access_log /var/log/nginx/access.log
       - error_log /var/log/nginx/error.log

    Reference patterns from RESEARCH.md "Pattern 3: Nginx Virtual Host Configuration".
  </action>
  <verify>
    <automated>docker compose run --rm nginx nginx -t 2>&1 | grep -q "syntax is ok" && echo "Nginx config valid"</automated>

<task type="auto">
  <name>Task 6: Create Makefile developer interface</name>
  <files>Makefile</files>
  <action>
    Create Makefile with the following targets:

    1. **up**: Start all containers in detached mode
       - docker compose up -d
       - Echo message: "Containers started. Access app at http://localhost"

    2. **down**: Stop all containers
       - docker compose down
       - Echo message: "Containers stopped."

    3. **logs**: View logs from all containers (follow mode)
       - docker compose logs -f

    4. **shell**: Shell into app container
       - docker compose exec app bash

    5. **shell-nginx**: Shell into nginx container (for debugging)
       - docker compose exec nginx sh

    6. **shell-mysql**: Connect to MySQL CLI
       - docker compose exec mysql mysql -u$(DB_USERNAME) -p$(DB_PASSWORD) $(DB_DATABASE)

    7. **install**: Run composer install in app container
       - docker compose exec app composer install

    8. **migrate**: Run database migrations
       - docker compose exec app php artisan migrate

    9. **seed**: Seed database
       - docker compose exec app php artisan db:seed

    10. **test**: Run PHPUnit tests
        - docker compose exec app php artisan test

    11. **build**: First-time setup (build + install + setup)
        - docker compose build
        - docker compose run --rm app composer install
        - Copy .env.example to .env if not exists
        - Generate application key
        - Run migrations
        - Echo completion message

    Add .PHONY declaration for all targets (prevents Makefile conflicts with files of same name).
    Set .DEFAULT_GOAL to help (add help target that lists all available commands).

    Reference patterns from RESEARCH.md "Pattern 4: Makefile Developer Interface".
  </action>
  <verify>
    <automated>make -n up 2>&1 | grep -q "docker compose up" && echo "Makefile up target valid"</automated>
  </verify>
  <done>Makefile exists with up, down, logs, shell, install, migrate, test, build targets</done>
